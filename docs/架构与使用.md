# 架构与使用

本文件说明项目结构、数据流、协议适配与排错。面向后续维护与二次开发。

## 模块划分

```
audit_core.py   核心：协议适配 + 探针 + 打分 + 横评（无第三方依赖）
webui.py        本地 WebUI：http.server 起服务，SSE 推进度（仅 127.0.0.1）
web/index.html  单页前端：表单 + 实时进度 + 对比结果（全内联，离线可用）
ztest_style_light_audit.py  CLI：默认逐站打分；--compare 取前两站横评
run_model_audit.py          CLI：官方端 vs 中转端对照（OpenAI 格式，未改动）
selftest.py     离线端到端自测：内置假中转，验证全链路（不花钱）
sites.example.json          站点配置模板（含 protocol 字段）
```

依赖关系：`webui.py`、`ztest_style_light_audit.py`、`selftest.py` 都只依赖 `audit_core.py`；`run_model_audit.py` 独立自洽。

## 核心数据流

```
site(协议/base_url/model/key) ─► Client(选 Adapter)
                                   │
                                   ├─ chat()   ─► 归一化响应 {text,id,model,usage,finish_reason,raw}
                                   └─ stream() ─► {content_type,chunks,done,text}

run_audit(site) ─► 跑 7 个探针 ─► 加权打分 + 封顶 ─► report{score,band,caps,model_echo,probes}

compare(A,B) ─► run_audit(A) + run_audit(B)
              + cross_consistency(A,B)  同一批确定性题 → 答案一致率/相似度/正确率/延迟
              + build_verdict()          推荐谁 + 关键差项 + 可疑信号
            ─► {report_a, report_b, cross, verdict, generated_at}
```

`compare` 接受 `progress` 回调，每完成一步推送一个 dict 事件，WebUI 用它做 SSE 实时进度。

## 协议适配

两个协议的差异都收敛在 `audit_core.py` 的 Adapter 里，探针逻辑与协议无关。

| | OpenAI | Anthropic |
|---|---|---|
| 端点 | `{base}/chat/completions` | `{base}/messages` |
| 鉴权头 | `Authorization: Bearer <key>` | `x-api-key: <key>` + `anthropic-version: 2023-06-01` |
| system | 作为 `messages` 里的 system 角色 | 顶层 `system` 参数 |
| max_tokens | 可选 | 必填（已强制带上） |
| 取文本 | `choices[0].message.content` | `content[] 里 type=text 的 text 拼接` |
| usage | `prompt_tokens/completion_tokens/total_tokens` | `input_tokens/output_tokens`（total 自算） |
| 结束原因 | `finish_reason` | `stop_reason` |
| 流式结束 | `data: [DONE]` | `event: message_stop` |
| 流式增量 | `choices[0].delta.content` | `content_block_delta` 的 `delta.text` |

新增协议：实现一个带 `endpoint/headers/build_body/parse_response/stream_delta/stream_done/error_bodies` 的 Adapter，注册进 `ADAPTERS` 即可，探针无需改动。

## 打分

7 个探针按权重加权（D1/D2/D17=22，D5=12，S3=12，D16=14，D8/D9=16，S5=10，S4=6）得到 0–100。
出现硬伤会**封顶**：协议签名 <40 封顶 40；system 锁定为 0 封顶 60；错误泄露 <70 封顶 70。
分数分档见 README「分数解读」。

横评结论 `build_verdict`：综合分差 ≥3 才分高下，否则判"基本持平"；分项差 ≥15 列入"关键差距"；
当确定性题一致率 <50% 且综合分差 ≥15，提示"疑似一方缩水或替换模型"；自报模型名不一致、任一方触发封顶也会作为可疑信号列出。

## WebUI 接口

| 方法 | 路径 | 作用 |
|---|---|---|
| GET | `/` | 返回 `web/index.html` |
| POST | `/api/run` | 提交 `{site_a, site_b, samples, probes}`，返回 `{job_id}` |
| GET | `/api/stream?job=<id>` | SSE 推送进度事件与最终 `result` |

事件类型：`stage` / `probe_start` / `probe_done` / `cross_start` / `cross_done` / `complete` / `result` / `error`。

安全：仅绑 `127.0.0.1`；key 仅在内存中用于本轮任务，不落盘、不写日志、不进 `results/`；任务结束后 job 即从内存移除。

## 排错

- **WebUI 打不开**：确认 `python webui.py` 正常打印了地址；换端口 `--port 9000`；浏览器手动访问 `http://127.0.0.1:8731/`。
- **某端全部探针失败**：多为 base_url 错误（少 `/v1`）、协议选错（OpenAI 站点选了 Anthropic）、key 失效或额度不足；展开"原始响应"看具体报错。
- **Anthropic 端报 max_tokens / system 相关错误**：确认填的是 Anthropic 兼容端点（`/messages`），且 base_url 到 `/v1` 为止。
- **流式探针 S5 为 0**：中转端可能不支持 SSE 或返回了非 `text/event-stream`，属于"流式假支持"信号。
- **想先验证工具本身是否正常**：跑 `python selftest.py`，看到 `ALL PASS` 即说明解析/打分/SSE/横评链路无问题。
